Flexbox

One Dimension

Flexbox is for one-dimensional layouts — arranging items in a row or a column. Think navbars, card rows, centering things. You set properties on the parent container, and the children respond.

↔️ flex-direction: row vs column

The most fundamental property. row (default) lays children horizontally. column stacks them vertically.

flex-direction: row
.container {
  display: flex;
  flex-direction: row; /* default */
}

⇔ justify-content — spacing along the main axis

Controls how items are distributed along the main axis (horizontal in row, vertical in column). Common values: flex-start, center, flex-end, space-between, space-around.

justify-content: space-between
.container {
  display: flex;
  justify-content: space-between; /* pushes items to edges */
}

💡 The navbar trick: Use justify-content: space-between with display: flex on a nav to push the logo left and links right — no floats needed!

⇕ align-items — cross axis alignment

align-items controls vertical alignment in a row (or horizontal in a column). center is the most powerful — it perfectly vertically centers items, something that was previously very hard in CSS.

align-items: center — items of different heights all centered
.container {
  display: flex;
  align-items: center; /* vertically centers everything */
  min-height: 100px;
}

↩ flex-wrap

By default, flex items try to squeeze into one line. flex-wrap: wrap lets them break onto new lines when they don't fit. Crucial for responsive layouts.

flex-wrap: wrap — items wrap to next line
.container {
  display: flex;
  flex-wrap: wrap;
  gap: 10px;

↔ flex-grow, flex-shrink, flex-basis

flex-grow: how much an item should grow to fill available space (0 = don't grow, 1 = grow). flex-shrink: how much to shrink when needed. flex-basis: starting size before growing/shrinking. Shorthand: flex: 1 = grow equally.

flex-grow: middle item fills remaining space
.middle { flex-grow: 1; } /* takes all leftover space */
.sides  { flex-grow: 0; } /* stays at natural size */

CSS Grid

Two Dimensions

Grid is for two-dimensional layouts — rows AND columns at the same time. Think page layouts, image galleries, dashboards. Flexbox handles a line of items; Grid handles the whole page structure.

▦ grid-template-columns

Defines how many columns and their widths. fr is the key unit — it means "fraction of available space". repeat(3, 1fr) = 3 equal columns.

3 equal columns
.grid {
  display: grid;
  grid-template-columns:
    repeat(3, 1fr);
  gap: 10px;
}

⬛ grid-column & grid-row (spanning)

Items can span multiple columns or rows. This is where Grid gets powerful — you can make some cells wider/taller than others. Use grid-column: span 2 to make an item take 2 columns.

Items spanning multiple columns
.hero   { grid-column: span 3; } /* full width */
.wide   { grid-column: span 2; } /* 2 of 3 cols */
.normal { grid-column: span 1; } /* default */

🗺 grid-template-areas — name your zones

The most readable way to define a layout. You literally draw your layout in ASCII, then assign elements to those named zones. Perfect for page-level structure.

Named grid areas — classic page layout
.page {
  display: grid;
  grid-template-areas:
    "header  header"
    "sidebar main  "
    "footer  footer";
  grid-template-columns: 160px 1fr;
  gap: 10px;
}

🧠 Flexbox vs Grid rule of thumb: Use Flexbox for components (navs, buttons, cards). Use Grid for page-level layouts (the overall structure of a page). They work great together — a Grid page with Flexbox components inside each area.
03

Responsive Design

Every Screen

Responsive design means your layout adapts to any screen size — phone, tablet, desktop. It's built on three pillars: fluid grids, flexible images, and media queries.

📱 Mobile-First Philosophy

Start with the mobile layout, then scale up. Write base CSS for small screens, then use media queries to add complexity for larger screens. This approach is cleaner and performs better on mobile.

Mobile-first approach
/* BASE — mobile first (no media query needed) */
.cards {
  display: flex;
  flex-direction: column; /* stacked on phone */
  gap: 16px;
}

/* TABLET — add 2 columns */
@media (min-width: 768px) {
  .cards { flex-direction: row; flex-wrap: wrap; }
  .cards > * { flex: 1 1 calc(50% - 8px); }
}

/* DESKTOP — 3 columns */
@media (min-width: 1024px) {
  .cards > * { flex: 1 1 calc(33% - 11px); }
}

💧 Fluid Units — %, vw, vh, clamp()

Instead of fixed px values, use relative units that scale with the viewport. clamp() is the modern hero: it sets a minimum, ideal, and maximum value all in one.

Fluid typography and sizing
/* OLD WAY — fixed px */
h1 { font-size: 48px; } /* too big on phone! */

/* NEW WAY — clamp(min, preferred, max) */
h1 { font-size: clamp(28px, 5vw, 64px); }
/* ↑ min 28px, scales with viewport, max 64px */

/* Fluid container */
.container {
  width: min(90%, 1200px); /* 90% wide but never over 1200px */
  margin: 0 auto;
}

✨ The Responsive Grid Trick — auto-fill + minmax

This one-liner creates a responsive grid with no media queries at all. Items automatically reflow based on available space. One of the most useful CSS patterns.

Fully responsive grid — zero media queries
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}
/* Items are at least 200px wide, as many as fit per row */
/* On phone: 1 col. Tablet: 2-3 cols. Desktop: 4-5 cols. */
⚡ auto-fill vs auto-fit: auto-fill creates empty columns if space allows. auto-fit collapses empty columns and stretches remaining items. Use auto-fit for card grids where you want items to stretch.

Media Queries

Breakpoints

Media queries let you apply different CSS at different screen sizes. They're the conditional logic of responsive design — "if the screen is this wide, apply this CSS."

📐 Common Breakpoints

These are the standard widths where layouts typically need to change. You don't have to use them all — start with just mobile + desktop, then add more as needed.

⌨ Syntax — min-width vs max-width

min-width = "at least this wide" (mobile-first). max-width = "up to this wide" (desktop-first). Use min-width when possible — it's the mobile-first way.

Media query syntax
/* min-width: mobile-first (recommended) */
@media (min-width: 768px) {
  .sidebar { display: block; } /* show sidebar on tablet+ */
}

/* max-width: desktop-first */
@media (max-width: 767px) {
  .sidebar { display: none; } /* hide sidebar on mobile */
}

/* Range — between two sizes */
@media (min-width: 768px) and (max-width: 1023px) {
  .nav { flex-direction: column; } /* tablet only */
}

/* Modern range syntax (2023+) */
@media (768px <= width <= 1023px) {
  /* tablet only */
}

🏗 Real-World Example — Nav & Layout

Here's what a typical responsive layout actually looks like in practice — a navbar that collapses, a grid that reflowes, and typography that scales.

Complete responsive component
/* ── Mobile base ── */
.nav {
  display: flex;
  flex-direction: column;
  gap: 8px;
}
.nav-links { display: none; } /* hidden on mobile */
.hamburger { display: block; }

.hero-title { font-size: clamp(28px, 6vw, 72px); }

.cards {
  display: grid;
  grid-template-columns: 1fr; /* 1 col */
  gap: 16px;
}

/* ── Tablet (768px+) ── */
@media (min-width: 768px) {
  .nav { flex-direction: row; justify-content: space-between; }
  .nav-links { display: flex; }
  .hamburger { display: none; }
  .cards { grid-template-columns: repeat(2, 1fr); }
}

/* ── Desktop (1024px+) ── */
@media (min-width: 1024px) {
  .cards { grid-template-columns: repeat(3, 1fr); }
}

🎨 Beyond Width — Other Media Features

Media queries can target much more than screen width. These modern features let you adapt to user preferences and device capabilities.

Other powerful media features
/* Dark mode preference */
@media (prefers-color-scheme: dark) {
  body { background: #000; color: #fff; }
}

/* Reduce motion (accessibility) */
@media (prefers-reduced-motion: reduce) {
  * { animation: none !important; }
}

/* Touch device / coarse pointer (finger) */
@media (pointer: coarse) {
  button { min-height: 44px; } /* bigger tap targets */
}

/* Orientation */
@media (orientation: landscape) {
  .hero { min-height: 60vh; }
}

/* Print styles */
@media print {
  .nav, .footer { display: none; }
  body { color: black; }
}
