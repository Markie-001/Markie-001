Chapter 01

Variables & Types

The building blocks of every program. Variables store data; types describe what kind of data it is.

1

Declaring Variables — let, const, var

Use const by default. Use let when the value will change. Avoid var — it's outdated.

javascript
const name = "Alice";        // can't be reassigned
let   score = 0;            // can be reassigned later
var   old = "avoid this";   // old way — has quirks

score = 10;   // ✓ works — let allows this
// name = "Bob"  ✗ ERROR — const can't change
2

The 7 Data Types

javascript — primitive types
const text    = "Hello";       // String
const age     = 25;            // Number (int or float)
const price   = 9.99;         // Number
const active  = true;         // Boolean
const nothing = null;         // Null (intentionally empty)
let   undef;                   // Undefined (not set yet)
const id      = Symbol("id"); // Symbol (unique)

// Check the type:
console.log(typeof text);   // "string"
console.log(typeof age);    // "number"
console.log(typeof active); // "boolean"
3

String Operations

javascript
const first = "John";
const last  = "Doe";

// Template literals (backticks) — the modern way
const full = `${first} ${last}`;   // "John Doe"
const msg  = `Hello, ${first}!`;  // "Hello, John!"

// Useful string methods
"hello".toUpperCase();  // "HELLO"
"  hi  ".trim();       // "hi"
"cat".includes("at");  // true
"abc".length;          // 3
💡

== vs ===

Always use === (strict equality). It checks value AND type. == does weird type coercion — "5" == 5 is true, which is almost never what you want.

Chapter 02

Functions & Scope

Functions are reusable blocks of code. Scope controls where variables can be accessed.

1

Three Ways to Write a Function

javascript
// 1. Function declaration (hoisted — usable before definition)
function greet(name) {
  return `Hello, ${name}!`;
}

// 2. Function expression (not hoisted)
const greet2 = function(name) {
  return `Hello, ${name}!`;
};

// 3. Arrow function (modern — most common today)
const greet3 = (name) => `Hello, ${name}!`;

// Call any of them the same way:
greet("Alice");   // "Hello, Alice!"
2

Parameters, Arguments & Default Values

javascript
// Parameters are the variable names in the definition
// Arguments are the actual values you pass in
const add = (a, b) => a + b;
add(3, 4);  // 7  — 3 and 4 are the arguments

// Default parameter values
const greet = (name = "stranger") => `Hi, ${name}!`;
greet();          // "Hi, stranger!"
greet("Bob");    // "Hi, Bob!"

// Functions can return anything
const double = n => n * 2;
double(5);   // 10
3

Scope — Where Can Variables Be Seen?

Variables declared with const or let only exist inside the block { } they were created in.

javascript — scope example
const globalVar = "I'm everywhere";  // global scope

function myFunc() {
  const localVar = "only inside here"; // function scope
  console.log(globalVar);  // ✓ can see global
  console.log(localVar);   // ✓ can see local
}

// console.log(localVar)  ✗ ERROR — localVar doesn't exist here

if (true) {
  const blockVar = "block scoped";
  // blockVar only lives inside this if block
}
// console.log(blockVar)  ✗ ERROR
4

Closures (Powerful!)

A function "remembers" the variables from the scope it was created in, even after that scope is gone.

javascript
function makeCounter() {
  let count = 0;                    // this variable is "remembered"
  return () => {
    count++;
    return count;
  };
}

const counter = makeCounter();
counter();  // 1
counter();  // 2
counter();  // 3  — count persists!

Chapter 03

DOM Manipulation

The DOM (Document Object Model) is JavaScript's interface to the HTML page. You can read, change, add, or remove any element.

🌳

What is the DOM?

When a browser loads HTML, it creates a tree of objects called the DOM. JavaScript can access this tree to read or change any element on the page — that's how dynamic websites work.

1

Selecting Elements

javascript
// querySelector — select ONE element (CSS selector syntax)
const title  = document.querySelector("h1");
const btn    = document.querySelector("#myButton");   // by id
const card   = document.querySelector(".card");      // by class

// querySelectorAll — select MULTIPLE elements → NodeList
const allBtns = document.querySelectorAll("button");
const items   = document.querySelectorAll(".item");

// Loop over results
items.forEach(item => console.log(item.textContent));
2

Reading & Changing Content

javascript
const el = document.querySelector("#myDiv");

// Read / write text
console.log(el.textContent);         // read
el.textContent = "New text!";       // write text

// Read / write HTML
el.innerHTML = "<strong>Bold!</strong>";

// Read / write CSS styles
el.style.color       = "red";
el.style.fontSize    = "24px";
el.style.display     = "none";    // hide

// Toggle CSS classes (best practice!)
el.classList.add("active");
el.classList.remove("active");
el.classList.toggle("active");    // add if missing, remove if present
el.classList.contains("active"); // true/false
3

Creating & Removing Elements

javascript
// Create a new element
const newItem = document.createElement("li");
newItem.textContent = "New list item";
newItem.classList.add("item");

// Add to the page
const list = document.querySelector("ul");
list.appendChild(newItem);     // add at end
list.prepend(newItem);         // add at start

// Remove an element
newItem.remove();              // removes itself

// Read/write attributes
const img = document.querySelector("img");
img.getAttribute("src");
img.setAttribute("alt", "A photo");

    Chapter 04

    Events & Listeners

    Events are things that happen in the browser — clicks, keypresses, mouse moves. Listeners let you react to them.

    1

    addEventListener — The Right Way

    javascript
    const btn = document.querySelector("#myBtn");

// Syntax: element.addEventListener(event, callback)
btn.addEventListener("click", () => {
  console.log("Button clicked!");
});

// Named function (easier to remove later)
const handleClick = () => console.log("clicked");
btn.addEventListener("click", handleClick);
btn.removeEventListener("click", handleClick); // remove it
    2

    The Event Object

    Every event listener receives an event object (e or event) with useful info about what happened.

    javascript
    btn.addEventListener("click", (e) => {
  console.log(e.target);          // the element that was clicked
  console.log(e.target.id);       // its id attribute
  console.log(e.type);            // "click"
  e.preventDefault();             // stop default behavior (e.g. form submit)
});

// Input events — reading what the user types
const input = document.querySelector("input");
input.addEventListener("input", (e) => {
  console.log(e.target.value);    // current text in input
});
    3

    4

    Event Delegation — One listener for many elements

    Instead of adding listeners to every child, listen on the parent and check e.target. Much more efficient!

    javascript
    // ❌ Inefficient — 100 listeners for 100 items
items.forEach(item => item.addEventListener("click", handler));

// ✅ Efficient — 1 listener on the parent
const list = document.querySelector("ul");
list.addEventListener("click", (e) => {
  if (e.target.matches("li")) {
    console.log("Clicked: " + e.target.textContent);
  }
});

    Arrays & Objects

    Arrays store ordered lists. Objects store named properties. Together they represent almost all real-world data.

    1

    Arrays — Ordered Lists

    javascript
    const fruits = ["apple", "banana", "cherry"];

fruits[0];          // "apple"  (zero-indexed!)
fruits.length;      // 3

// Add / remove
fruits.push("mango");   // add to end   → ["apple","banana","cherry","mango"]
fruits.pop();           // remove last  → ["apple","banana","cherry"]
fruits.unshift("kiwi"); // add to front
fruits.shift();          // remove first

// Check if item exists
fruits.includes("banana");   // true
fruits.indexOf("cherry");    // 2
    2

    The 4 Array Superpowers — map, filter, find, reduce

    javascript — these are used constantly in real code
    const nums = [1, 2, 3, 4, 5];

// map → transform every item → returns NEW array
const doubled = nums.map(n => n * 2);   // [2,4,6,8,10]

// filter → keep items that pass a test → returns NEW array
const evens = nums.filter(n => n % 2 === 0); // [2,4]

// find → get the FIRST match (single item)
const big = nums.find(n => n > 3);    // 4

// reduce → collapse array to a single value
const total = nums.reduce((sum, n) => sum + n, 0); // 15

// forEach → loop without returning (for side effects)
nums.forEach(n => console.log(n));
    3

    Objects — Named Properties

    javascript
    const user = {
  name:  "Alice",
  age:   28,
  email: "alice@example.com",
  greet() {                           // method inside object
    return `Hi, I'm ${this.name}`;
  }
};

// Access properties
user.name;           // "Alice"  — dot notation
user["age"];         // 28      — bracket notation (useful for dynamic keys)

// Add / change / delete
user.city   = "Amsterdam";    // add new property
user.age    = 29;              // update
delete user.email;             // remove

// Destructuring — pull out values elegantly
const { name, age } = user;
console.log(name, age);        // "Alice" 29
    4

    Arrays of Objects — Real-world Data

    javascript — this is how APIs return data
    const users = [
  { id: 1, name: "Alice", active: true  },
  { id: 2, name: "Bob",   active: false },
  { id: 3, name: "Carol", active: true  },
];

// Get only active users
const active = users.filter(u => u.active);

// Get just the names
const names = users.map(u => u.name);  // ["Alice","Bob","Carol"]

// Find a specific user
const bob = users.find(u => u.id === 2);
